# Future Improvements for Valiant Picks

## High Priority (User Experience)

### 1. Add Loading Skeletons
**Current**: Shows "Loading..." text
**Better**: Skeleton screens for better perceived performance
**Benefit**: Makes the app feel faster, better UX during data fetching
**Implementation**: Add skeleton components for dashboard cards, game lists, leaderboard rows

### 2. Debounce Bet Amount Input
**Issue**: Validation runs on every keystroke while typing
**Solution**: Debounce validation by 300-500ms
**Benefit**: Reduces unnecessary re-renders, smoother typing experience
**Files**: Dashboard.js, Games.js

### 3. Add Bet Confirmation Dialog
**Purpose**: Prevent accidental high-value bets
**Features**: 
  - Show bet summary (team, amount, potential win)
  - Require explicit confirmation
  - Highlight for bets over $100
**Benefit**: Prevents costly user mistakes, adds confidence
**Priority**: HIGH - prevents user errors

### 4. Better Error Messages
**Current**: Generic error messages
**Better**: User-friendly, actionable error messages
**Examples**:
  - "Insufficient balance" â†’ "You need $X more to place this bet"
  - "Game locked" â†’ "Betting closed - game starts in 5 minutes"
  - "Server error" â†’ "Connection issue - your bet was not placed"

### 5. Add "Recent Winners" Section
**Location**: Dashboard top section
**Display**: Recent winning bets from all users
**Features**: 
  - Username, team, amount won
  - Sliding carousel or auto-scroll
  - Creates FOMO and social proof
**Benefit**: Increases engagement, shows active community

### 6. Keyboard Navigation
**Add shortcuts**:
  - Enter: Submit active form
  - Esc: Close modals/dialogs
  - Tab: Better focus management
  - Arrow keys: Navigate between games
**Benefit**: Power users can bet faster, better accessibility

### 7. Add Bet Slip Preview (Real-time Calculation)
**Feature**: Live calculation as user types bet amount
**Display**: 
  - Amount wagered
  - Confidence multiplier
  - Potential payout
  - Profit if won
**Location**: Next to bet form, updates in real-time
**Benefit**: User knows exact payout before confirming
**Priority**: HIGH - improves clarity

### 8. Implement Rate Limiting on Client
**Issue**: User can accidentally double-click "Place Bet"
**Solution**: 
  - Disable button after click
  - Show loading state
  - Re-enable only after response or timeout
**Benefit**: Prevents double-betting bugs
**Priority**: HIGH - prevents critical bugs
